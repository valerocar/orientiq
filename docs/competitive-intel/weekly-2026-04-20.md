## 1. arXiv / Research Papers

### [2604.12236] Multi-Axis Additive Manufacturing for Customized Automotive Components
**What:** Multi-axis DLP 3D printing via robotic arm (University of Michigan-Dearborn, April 14 2026). The robot reorients the build platform mid-print to cure at varying angles, eliminating many support structures without pre-print orientation planning.  
**Why it matters:** Orthogonal threat vector — instead of finding the best fixed gravity direction, it dynamically changes the build axis per layer. For complex automotive geometry this can outperform any single-orientation solver including orientIQ. Still hardware-dependent (robotic arm + DLP), so not a near-term slicer-level competitor, but worth tracking as a research direction that could eventually make static orientation optimization less relevant for high-end parts.  
**Source:** https://arxiv.org/abs/2604.12236

### Topology + Build Orientation Co-Optimization (Springer, 2025, surfacing this week)
**What:** "Combined topology and build orientation optimization for support structure minimization" and "Comprehensive additive manufacturing cost reduction through simultaneous topology and build orientation optimization" — both appearing in search results with fresh indexing.  
**Why it matters:** These co-optimize part *shape* and orientation jointly, which is out of scope for orientIQ (we take geometry as fixed). Not a direct threat, but reinforces that build-direction optimization is an active research priority. Validates the problem space.  
**Sources:** https://link.springer.com/article/10.1007/s00158-025-04124-6 · https://link.springer.com/article/10.1007/s40964-025-01520-8

**No papers found** this week specifically using Hopf fibration, Gauss map, or Riemannian gradient descent in the context of print orientation. orientIQ's mathematical framing remains differentiated in the literature.

## 2. Slicer Releases

### Autodesk Netfabb 2026 — Released
**What:** Netfabb 2026 ships with a redesigned UI surfacing orientation and support tools more prominently, new control over support orientation relative to the horizontal plane, improved nesting via "True Shape" packing for powder-bed fusion, and enhanced cluster detection for anisotropic recoater effects.  
**Why it matters:** Netfabb is the enterprise incumbent for metal AM prep. The orientation UI improvements and True Shape packing narrow the gap for professional users who previously needed a standalone tool. orientIQ's advantage remains: multi-objective Pareto exploration, Hopf-fibration math, and open Python API — none of which Netfabb exposes. Monitor whether future releases add Pareto-style trade-off visualization.  
**Sources:** https://3dprintingindustry.com/news/autodesk-updates-netfabb-with-enhanced-workflow-tools-and-new-3mf-support-in-2026-release-238996/ · https://forums.autodesk.com/t5/netfabb-forum/autodesk-netfabb-2026-is-now-available/td-p/13437539

### Cura 3.0 — Pre-release, Spring 2026 window
**What:** Cura 3.0 is in pre-release; no confirmed auto-orient feature changes found this week. Existing orientation support is still the community marketplace plugin (heuristic, not gradient-based).  
**Why it matters:** Low threat at present. If 3.0 ships a native algorithmic auto-orient, that would be material — check release notes when it drops.  
**Source:** https://3dput.com/best-3d-printer-slicer-software-2026-cura-prusaslicer-bambu-studio-orca-slicer/

## 3. Startups

### Castor — Liquidated (2025)
**What:** Israeli AM software startup Castor (geometry-based part feasibility + orientation tooling) filed for court-ordered liquidation in 2025 after funding dried up amid a downturn in US 3D printing markets.  
**Why it matters:** Removes one direct software competitor in the "geometry-aware AM optimization" category. Their Oqton/Materialise CO-AM integrations are now orphaned. Potential opening to position orientIQ with customers who relied on Castor's orientation and feasibility features.  
**Source:** https://3dprint.com/319042/israeli-3d-printing-software-startup-castor-files-for-liquidation/
